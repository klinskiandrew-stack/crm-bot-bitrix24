# Telegram-бот для Bitrix24 CRM с ИИ (Ассистент Гроу)

Telegram-бот, который выступает интеллектуальным интерфейсом к CRM Bitrix24. Партнёры добавляют бота в групповые чаты, упоминают его в вопросах, и бот отвечает, используя Claude AI (через Kie.ai) для обработки данных из Bitrix24.

## Возможности

- **Естественный язык интерфейс** — задавайте вопросы о сделках, лидах, активности в естественном языке
- **ИИ-powered анализ** — Claude анализирует данные CRM и дает рекомендации
- **Безопасность данных** — read-only доступ, изоляция данных между партнёрами
- **Маршрутизация моделей** — автоматический выбор подходящей модели Claude (Haiku → Sonnet → Opus)
- **Оптимизация стоимости** — prompt caching снижает стоимость запросов на 50%+
- **Полный аудит** — логирование всех запросов с метриками токенов и расхода credits

## Установка

### Требования
- Python 3.11+
- SQLite3
- Telegram Bot Token (от BotFather)
- Bitrix24 API Webhook URL
- Kie.ai API Key

### Шаги установки

1. Клонируем репозиторий:
```bash
git clone <repo_url>
cd crm-bot
```

2. Создаём виртуальное окружение:
```bash
python3.11 -m venv venv
source venv/bin/activate  # на Windows: venv\Scripts\activate
```

3. Устанавливаем зависимости:
```bash
pip install -r requirements.txt
```

4. Копируем и заполняем `.env`:
```bash
cp .env.example .env
```

Редактируем `.env` с вашими credentials:
- `TELEGRAM_BOT_TOKEN` — токен от BotFather
- `ADMIN_TELEGRAM_ID` — ваш Telegram ID (для админ-панели)
- `B24_WEBHOOK_URL` — webhook URL для вашего портала Bitrix24
- `KIE_API_KEY` — ключ API Kie.ai

5. Инициализируем базу данных:
```bash
python scripts/init_db.py
```

6. Запускаем бота:
```bash
python main.py
```

## Использование

### В групповом чате
Упомяните бота и задайте вопрос:
```
@crm_bot сколько сделок в работе?
@crm_bot проанализируй активность Ивана за неделю
@crm_bot какой статус лида от компании "Рога и копыта"?
```

### Админ-команды
- `/admin` — открыть админ-панель (только для владельца)
- `/set_model <model>` — выбрать модель (haiku, sonnet, opus)
- `/stats` — базовая статистика использования

## Этапы развития

### Этап 1 (MVP) — Базовая функциональность
- ✓ Структура проекта
- ✓ Telegram-бот с прокси-поддержкой
- Интеграция с Bitrix24 (2-3 метода)
- Первый вызов Claude
- Базовые админ-команды

### Этап 2 — Полная функциональность
- Function calling с инструментами
- Orchestrator цикл
- Маршрутизация моделей
- Полная авторизация и фильтрация
- Админ-панель с inline-кнопками

### Этап 3 — Оптимизация
- Prompt caching
- Метрики и дашборд
- Обработка ошибок

### Этап 4 — Партнёрский интерфейс
- Личные меню для партнёров
- Шаблонные отчёты
