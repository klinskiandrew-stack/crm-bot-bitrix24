# AI-ассистент CRM — функциональная карта

Документ для составления коммерческого предложения. Описывает, что
умеет ассистент в продакшене (на примере проекта Growzone). Все
возможности проверены в реальной работе.

---

## Что это в одной фразе

Telegram-бот, который интегрирован с CRM компании, телефонией,
финансовой отчётностью и рекламными кабинетами — и отвечает на вопросы
бизнеса на естественном языке (текстом и голосом) за секунды, без
необходимости открывать ни одну из подключённых систем.

Заменяет нескольких специалистов в одном чате — аналитика, РОПа,
финдиректора, маркетолога, аудитора звонков.

---

## БЛОК 1 — Интерфейс

### 1.1 Telegram как единая точка входа
- Бот в Telegram-чате (рабочий чат компании или личка).
- Триггеры: упоминание `@bot`, ответ на сообщение бота, любое сообщение
  в личке.
- Никаких отдельных интерфейсов, дашбордов, новых программ для команды.

### 1.2 Голосовые команды
- Любое голосовое сообщение или видео-кружок → распознаётся локальным
  движком Whisper → обрабатывается как текстовый вопрос.
- Распознавание полностью на сервере клиента — **без поминутной оплаты
  сторонним сервисам STT** (Yandex SpeechKit, Google и т.п. не нужны).

### 1.3 Память диалога
- Помнит контекст разговора — можно уточнять, не повторяя.
- Прогресс-индикация: показывает каждый шаг («думаю», «получаю сделки»,
  «считаю»).

### 1.4 Изоляция данных по ролям
- Каждый сотрудник видит только свою часть CRM.
- Партнёр / РОП / админ — разный объём доступа.
- Гибкая настройка: можно ограничить по конкретным менеджерам или
  отделам.

## БЛОК 2 — Аналитика CRM (Bitrix24, можно адаптировать под другие)

### 2.1 Свободные вопросы про CRM
«Сколько лидов с Авито в мае?», «Покажи воронку», «Что по сделке N?»,
«Какие сделки в работе на стадии замера?» — бот сам идёт в CRM, считает
и отвечает за секунды.

### 2.2 Глубокая аналитика
- **Точные агрегаты** по лидам и сделкам: count + сумма + разбивки по
  источникам, направлениям, менеджерам, стадиям — в одном запросе.
- **Анализ причин отказов** лидов и сделок: группировка по причинам с
  примерами карточек.
- **Конверсия** между этапами воронки (созданные → замер → договор → продажа).
- **Прогноз продаж** на месяц на основе текущего темпа.

### 2.3 Аналитика по менеджерам
- Активность каждого менеджера (звонки, встречи, задачи, новые сделки).
- Скорость реакции на лид (speed-to-lead).
- Конверсия каждого менеджера.
- Сравнение менеджеров между собой.
- Бот знает поимённый состав отдела продаж и фильтрует «своих» от других
  сотрудников.

## БЛОК 3 — Контроль качества звонков (2 модуля)

### 3.1 Звонки колл-центра / перехвата
- Бот слушает рабочий чат КЦ, ловит онлайн-отчёты по каждому звонку.
- Скачивает MP3, расшифровывает локальным Whisper.
- AI-аналитик оценивает оператора **по утверждённому скрипту** (а не
  своим выдуманным критериям): оценка 1-5, что сделал по скрипту, что
  упустил, температура лида.
- Связывает звонок с лидом/сделкой в CRM.
- Складывает всё в общую Google-таблицу — сквозная аналитика «звонок
  → сделка → результат».
- Прогресс-уведомления в чат КЦ по каждому звонку в реальном времени.

### 3.2 Звонки менеджеров с клиентами (Mango Telecom / любая телефония)
- По запросу «разбери звонки по лиду N» — бот достаёт все записи из
  таймлайна карточки CRM, скачивает их через API, расшифровывает,
  оценивает по регламенту менеджера.
- На выходе: оценка каждого звонка 1-5 с конкретным разбором, со
  ссылками на пункты регламента («не задал вопрос 4.3», «не отработал
  возражение 7.1»).
- Кеширование транскриптов — повторный разбор того же звонка мгновенный.
- Регламент хранится в одном файле, который заказчик может править без
  программиста — изменения применяются на следующем запросе.

**Применение:**
- РОП за минуту получает разбор работы менеджера по конкретной сделке.
- Контроль качества колл-центра без ручного прослушивания.
- Видно типовые ошибки: «менеджеры не дожимают на замер», «никто не
  закрывает возражение по цене».

## БЛОК 4 — Радар продаж (проактивный поиск денег)

Бот не ждёт вопроса — каждый понедельник сам присылает руководителю
дайджест «Идеи для продаж»:

- **Замеры без продолжения** — замер сделан (выезд инженера оплачен!),
  но сделка стоит. Деньги в шаге от закрытия.
- **Застрявшие сделки** — в работе 14+ дней без движения по стадии.
- **Забытые лиды** — активные лиды без касаний 2+ дней.
- **Новые лиды без реакции** — созданы, но менеджер не взял в работу за
  4+ часа.

По каждому пункту: сумма ₽, менеджер, сколько простаивает, ссылка на
карточку.

**На реальных данных Growzone модуль за один прогон подсветил почти
18 млн ₽ в сделках с готовым замером, где продажа просто зависла.**

Параллельно — те же данные доступны по запросу («где зависли сделки»,
«что дожать», «кого забыли»).

## БЛОК 5 — Финансы (CFO-режим)

- Подключение к управленческой отчётности заказчика в Google Sheets
  (ЛУС, ОПиУ, P&L).
- Бот понимает терминологию: выручка план / признанная, маржинальная
  прибыль, рентабельность, дебеторка.
- Вопросы: «прибыль за май», «рентабельность по проектам в работе»,
  «сравни месяцы», «топ статей расходов».
- Ответ — в финансовой лексике, не выгрузкой цифр.

## БЛОК 6 — Маркетинг и трафик

- **Яндекс.Метрика** — трафик, источники, конверсии по целям.
- **Avito** — баланс, объявления, статистика, расходы, слабые объявления
  («много показов, ноль обращений»).
- **Яндекс.Директ** — модуль готов, активируется по апруву API.
- Сквозная аналитика канала: расход → лиды → сделки → выручка.

---

## БЛОК 7 — Сервисные функции

- **Выгрузка лидов в Excel** прямо в чат — с UTM-метками, телефонами,
  причинами отказов (для директолога/аналитика).
- **Планирование Zoom-встреч** из чата — голосование по слотам или
  прямое создание встречи.
- **Авто-мониторинг ошибок** самого бота — ежедневный дайджест проблем
  админу.
- **Ежедневные/недельные/месячные отчёты** по продажам в рабочий чат
  (cron, настраивается).

## БЛОК 8 — Что под капотом (для технической части КП)

- **LLM:** DeepSeek V4-Flash напрямую — стоимость одного запроса
  ~0.001-0.01 ₽. Все ограничения по бюджету настроены (дневной лимит
  расходов, тайм-аут на запрос).
- **Распознавание речи:** локальный Whisper (faster-whisper, модель
  small, int8). Никаких сторонних подписок.
- **Сервер:** VPS 4 ядра / 4 GB RAM / 50 GB SSD — этого хватает.
  Деплой систем-сервисом, автоперезапуск, мониторинг.
- **Базы данных:** SQLite для бот-метаданных (логи запросов, кэш,
  сессии), внешние API — для данных.
- **Безопасность:** доступ к CRM только на чтение, изоляция данных по
  пользователям, аудит-лог каждого запроса.

---

## БЛОК 9 — Что доступно к расширению (продаётся как этап 2)

- **Анализ переписок в Открытых линиях** (Avito-чат, WhatsApp, Telegram,
  чат сайта) — оценка скорости ответа менеджера + качества отработки
  возражений в чате (модуль готов архитектурно, нужно подключить scope).
- **Реактивация потерянных лидов** — список «дорого/думает/не сезон» на
  перезвон с сезонной логикой.
- **Cross-sell по завершённым клиентам** — кто купил X, не брал Y.
- **Веб-дашборд** — интерфейс для тех, кто не пользуется Telegram.
- **Партнёрский интерфейс** в личных сообщениях с кнопочным меню.

## БЛОК 10 — Ценностные тезисы для КП

- **Скорость:** ответ за секунды вместо 15-30 минут на ручную выгрузку.
- **24/7:** работает в нерабочее время, отвечает с телефона.
- **Голос:** можно спрашивать на ходу, не открывая ноутбук.
- **Объективность:** оценка звонков менеджеров по утверждённому
  регламенту, а не «по ощущениям» РОПа.
- **Проактивность:** сам находит сделки, которые могут утечь.
- **Команда в одном лице:** аналитик + РОП + финдиректор + аудитор
  звонков + маркетолог в одном чате.
- **Адаптируется под бизнес:** регламент звонков, состав отдела продаж,
  направления услуг, источники трафика — всё конфигурируется.
- **Дешёво в эксплуатации:** ~3–5 тыс ₽/мес на LLM, сервер 1.5 тыс ₽/мес,
  без подписок на STT.

## Что НЕ умеет (честно для КП, чтобы не было ложных ожиданий)

- Не пишет за менеджера сообщения клиентам автоматически (это
  доделывается, но требует решения по тону / ответственности).
- Не выгружает контент Базы знаний Bitrix через API — это ограничение
  платформы Bitrix, обходится копипастом.
- Не интегрируется с произвольной CRM из коробки — каждая CRM требует
  адаптации клиента (API, схема данных, права).
- Не заменяет менеджеров — он усиливает их и контролирует, но
  переписку/звонки ведут люди.

## Типовая структура внедрения (для оценки сроков в КП)

1. **Аудит CRM и систем заказчика** — 3-5 дней.
2. **Базовый MVP** (CRM-аналитика + Telegram-бот) — 2-3 недели.
3. **Голосовые команды + первая интеграция аналитики** (Метрика / Avito) — +1 неделя.
4. **Модуль контроля звонков** (если есть телефония с записями) — 1-2 недели.
5. **Радар продаж** (если есть осмысленная воронка) — 1 неделя.
6. **Финансовый блок** (если есть управленческая отчётность) — 1 неделя.
7. **Кастомные регламенты, обучение команды, передача** — 1 неделя.

Итого base: ~6-8 недель до промышленной эксплуатации.
