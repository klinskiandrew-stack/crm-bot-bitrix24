⚡ БЫСТРЫЙ СТАРТ (скопируйте поочередно)
════════════════════════════════════════════════════════════════════════════════

# 1. Подключиться к серверу
ssh root@YOUR_SERVER_IP

# 2. Установить ПО
apt update && apt install -y python3.11 python3.11-venv git sqlite3
useradd -m -s /bin/bash crmbot
su - crmbot

# 3. Клонировать проект
cd /opt
git clone <REPO_URL> crm-bot
cd crm-bot

# 4. Создать окружение
python3.11 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
mkdir -p data logs

# 5. Создать .env файл (заполните переменные!)
cat > .env << 'EOF'
TELEGRAM_BOT_TOKEN=ВАШ_ТОКЕН
ADMIN_TELEGRAM_ID=ВАШ_ID
B24_WEBHOOK_URL=https://ваш-портал.bitrix24.ru/rest/1/token/
KIE_API_KEY=ВАШ_KIE_API_KEY
KIE_BASE_URL=https://api.kie.ai/claude/v1
DATABASE_PATH=./data/bot.sqlite
LOG_LEVEL=INFO
EOF

chmod 600 .env

# 6. Инициализировать БД
python3 scripts/init_db.py

# 7. Протестировать локально
python3 main.py
# (Ctrl+C для выхода если всё OK)

# 8. Создать systemd сервис
exit  # выйти от crmbot

sudo bash -c 'cat > /etc/systemd/system/crm-bot.service << EOF
[Unit]
Description=CRM Bot
After=network.target

[Service]
Type=simple
User=crmbot
WorkingDirectory=/opt/crm-bot
Environment="PATH=/opt/crm-bot/venv/bin"
ExecStart=/opt/crm-bot/venv/bin/python3 /opt/crm-bot/main.py
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal
SyslogIdentifier=crm-bot

[Install]
WantedBy=multi-user.target
EOF'

sudo systemctl daemon-reload
sudo systemctl enable crm-bot
sudo systemctl start crm-bot

# 9. Проверить статус
sudo systemctl status crm-bot

# 10. Просмотреть логи
sudo journalctl -u crm-bot -f

⚠️  ЕСЛИ ЧТО-ТО НЕ РАБОТАЕТ
════════════════════════════════════════════════════════════════════════════════

Бот не запускается:
  1. Проверьте логи: sudo journalctl -u crm-bot -n 50
  2. Проверьте .env: cat /opt/crm-bot/.env
  3. Проверьте синтаксис: cd /opt/crm-bot && python3 -m py_compile main.py

Нет ответов от бота:
  1. Убедитесь, что бот добавлен в группу
  2. Убедитесь, что сервис запущен: sudo systemctl status crm-bot
  3. Проверьте доступ к Telegram: curl https://api.telegram.org/
  4. Проверьте credentials в .env

Медленная работа:
  1. Проверьте размер БД: du -h /opt/crm-bot/data/bot.sqlite
  2. Запустите healthcheck: cd /opt/crm-bot && source venv/bin/activate && python3 scripts/healthcheck.py
  3. Очистите старые логи (> 90 дней)
